## Not Changed (Documented for Future Reference)

These items were identified but intentionally left alone to avoid behavioral changes:

- **State duplication** between `shared.js`, `chat-bubble.js`, and `interactive.js` (email, company, answers, sessionId). Each module reads from localStorage independently. Consolidating would require an event-based sync system — out of scope for a quality pass.
- **Silent `catch` blocks** in `dynamic-stats.js`, `chat-bubble.js`, `interactive.js`. These intentionally fall back to defaults when the API or stats.json is unavailable. Adding console warnings would be a behavior change.
- **countUp animation** exists in three slightly different forms across `charts.js`, `interactive.js`, and `shared.js`. Each has different easing, duration, and formatting needs — not true duplicates.
- **Inline style manipulation** in page transitions, card tilt, and magnetic buttons (`shared.js`). These are runtime-computed values (mouse position, scroll position) that can't be moved to CSS classes.
