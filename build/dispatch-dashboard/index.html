The Pipeline

Every automated task follows the same five-stage path. No human in the loop once a task is dispatched. The intelligence lives in execution, not routing.

Stage 1
VPS Claw
Orchestrator on srv1354778. Reads task queue, builds HMAC-signed manifests, dispatches over SSH.
Stage 2
Paula
Routing layer. Validates signatures, checks plugin registry, routes to the correct executor.
Stage 3
Claude Code
Execution engine. Reads context, reasons through steps, writes code, runs tests. Flat-rate via Max subscription.
Stage 4
GitHub
Commits pushed to target repo. Branch strategy, PR creation, and merge handled automatically.
Stage 5
Cloudflare
Pages deploy triggers on push. Site live within seconds. Workers updated via wrangler.

How It Works

  1. VPS Claw reads the task queue. A SQLite database on the VPS holds pending tasks with type, priority, and target repo. Cron checks every 5 minutes. Tasks can also be triggered by webhooks or manual dispatch.
  2. Paula validates and routes. Each task manifest is HMAC-SHA256 signed. Paula verifies the signature, checks the plugin registry for backend affinity, and forwards to the correct executor. No LLM in the routing path.
  3. Claude Code executes. The task lands as a prompt with full context: files to read, changes to make, constraints to follow. Claude Code does the reasoning, writes code, runs validation. All on the Max subscription.
  4. GitHub receives the commit. Claude Code pushes directly or creates a PR depending on the task config. Commit messages follow the repo's conventions. Branch protection rules apply normally.
  5. Cloudflare deploys. For website repos, Cloudflare Pages triggers on push. For worker repos, wrangler deploys the updated worker. The pipeline end-to-end takes under 2 minutes for most tasks.
