# Briu Messaging Strategy

Source: Lucas brain dump, March 14 2026

## The core thesis

The AI subscriptions you already pay for to think are the same ones that build your software. There is no separate dev cost. Ideas become working tools at near-zero incremental cost.

## The three layers

### Layer 1: Reason ($20/month per person)
Any leading model chat. Claude Pro, ChatGPT Plus. Brain dumps via voice-to-text, strategy, analysis, research. Every person on the team starts here. This is the gateway.

### Layer 2: Build ($200-400/month per build station)
Claude Max + ChatGPT Pro on one device. Claude Code and Codex turn ideas into working software. Same subscription used for reasoning also writes code, builds websites, runs security audits, deploys tools. Zero incremental cost per task.

### Layer 3: Orchestrate (~$400-800/month)
A lightweight VPS agent that dispatches, monitors, and coordinates. The only variable API cost. Signs tasks, tracks completion, logs everything. Your always-on operations layer.

## The pitch to businesses

1. Your whole team learns to use AI for reasoning (Layer 1). Everyone gets a subscription, starts using it daily for their actual work.

2. Your most technical person becomes the agent builder. They use Layer 2 to build tools the whole team needs. Custom, integrated, specific to your business.

3. Over time, the code your agents write runs programmatically. Scripts on cron, automations, pipelines. Your dependency on the AI decreases as the codebase grows. You're building equity, not renting capability.

4. Software subscriptions are dead. Tell your SaaS provider you have a Claude Code subscription and you're considering building it in-house. Your bills should drop 80%. You get software that's fully integrated, specific to your business, and improvable by your own team.

5. Within a year, open source models will match today's frontier models. At that point you run agents on local hardware. The system you built now works with free models later.

## What Briu does

- Assessment: come in, understand the business, identify the highest-value agent implementations
- Training: teach the team to use AI for reasoning, teach the technical lead to build with agents
- Implementation: set up the infrastructure (VPS, subscriptions, dispatch system, monitoring)
- Handover: the client owns everything. No recurring Briu dependency. They keep their subscriptions running.

- Not an AI SaaS product (not selling software, selling implementation expertise)

## The scaling story for clients

One person with one device: handles email, CRM, reporting, basic automation
One builder with build subscriptions: creates custom tools for the whole team
Full agent stack: company-level agents, team agents, personal agents, all dispatching
End state: every person on the team dispatches work to agents, the technical lead maintains the system, SaaS costs go to near-zero, everything is owned

## Why transparency matters

Briu documents everything publicly. The /build page shows real costs, real tasks, real code output. This proves competence and builds trust. A client can see exactly what the system does before buying. No sales deck needed. The product IS the proof.
